# FP16 PTX → Default (with FP16_WMMA Legacy)

**작성일**: 2026-06-08
**목적**: FP16 trailing GEMM 기본 경로를 WMMA → PTX (`mma.m16n8k16`) 로 전환. tf32 가 이미 PTX 디폴트인 것과 대칭. WMMA 경로는 `fp16_wmma` 로 노출되어 비교 측정용으로 보존.
**적용 파일**: `src/factorize/{dispatch,kernels,phases}.cuh`, `src/multifrontal.{cu,hpp}`, `tests/run_custom_solver.cu`
**측정 GPU**: RTX 3090, CUDA 12.8

## 배경

지금까지 `--precision fp16` 은 FP16 WMMA m16n16k16 + Csc fragment readback 패턴을 사용했다. TF32 영역에서는 이미 docs/15 (V9h) 에서 PTX 가 권장 default 가 됐고 (`--precision tf32 = PTX`, `--precision tf32_wmma = legacy`), 같은 패턴을 FP16 에 적용.

## 변경 요약

### enum + CLI mapping

| CLI | enum | 산술 / 의미 |
|---|---|---|
| `--precision fp16` | `Precision::FP16` | **FP16 PTX `mma.m16n8k16`, FP32 accumulate, no Csc (default)** |
| `--precision fp16_wmma` | `Precision::FP16_WMMA` | FP16 WMMA m16n16k16 + Csc readback (legacy) |
| `--precision tf32` | `Precision::TF32` | TF32 PTX V9h (default, no change) |
| `--precision tf32_wmma` | `Precision::TF32_WMMA` | TF32 WMMA V0 (legacy, no change) |

### 새로 추가된 device code

`src/factorize/phases.cuh`:
- `pack_half2_bits(__half lo, __half hi) → unsigned`
- `trailing_update_mma_fp16_ptx(...)` — inline PTX `mma.sync.aligned.m16n8k16.row.col.f32.f16.f16.f32`, A-reuse loop (NTJ8_MAX=8 N-tile chunks), 4 per-lane FP32 accumulators 직접 `F` 에 빼기 (Csc 없음).

`src/factorize/kernels.cuh`:
- `factor_mid_fp16_ptx(...)` — mid tier `Precision::FP16` 전용. 256-thread 블록까지, KP=round_up(nc, 16).
- `factor_big_fp16_ptx(...)` — `__launch_bounds__(512, 2)` big tier. tf32 PTX big 과 동일 launch shape.

### dispatch 변경

- mid 단계: `Precision::FP16` → `factor_mid_fp16_ptx`, `Precision::FP16_WMMA` → 기존 `factor_mid_tc` (legacy).
- big 단계: 같은 패턴, `Precision::FP16` → `factor_big_fp16_ptx`, `Precision::FP16_WMMA` → 기존 `factor_big_tc`.
- 10pct (`factorize-b1-10pct-codex` 통합) 의 `fp16_b1_scalar_mid` / `fp16_scalar_spine` 분기는 **두 FP16 모드 모두에 동일 적용** (B=1 small-n 에서 mid kernel 회피).

### `src/multifrontal.cu`

`cudaFuncSetAttribute(..., MaxDynamicSharedMemorySize, 99 KB)` 등록을 `factor_mid_fp16_ptx`, `factor_big_fp16_ptx` 에 추가. (docs/15 의 영구 학습 v: 새 `__global__` 등록 누락 시 graph capture mode 에서 segfault.)

## 측정

ACTIVSg10k B=64 batched factor per system (median of 3 trial × 100 rep):

| Mode | per-sys ms |
|---|---|
| `fp16` (PTX default) | **0.0321** |
| `fp16_wmma` (legacy) | 0.0348 |

→ PTX 가 WMMA legacy 대비 −7.7%. B=1 단일 케이스에서도 동급 또는 약간 빠름. residual 양쪽 모두 baseline 수준 (~1e-4 FP16 정밀도).

## 이전 작업과의 관계

이 변경은 다음 두 작업 위에 stack 됨:
- docs/21 의 솔브 spine fusion + multi-stream
- `perf/factorize-b1-10pct-codex` 워크트리의 10pct factor B=1 가속 (factor_small_single, factor_panel_chain_staged_single, do_extend & 2 plain-extend bit)

원본 stash (`stash@{1}` 의 codex 작업) 에는 `Precision::TC` 하이브리드 모드 (mid TF32 PTX + big FP16 PTX) 도 포함됐으나 사용자 결정으로 **FP16_PTX 만 수령, TC 모드는 폐기**.

## 측정 재현

```bash
# B=1 fp16 PTX vs WMMA legacy
for prec in fp16 fp16_wmma; do
  ./build/custom_linear_solver_run \
    --matrix /datasets/power_system/nr_linear_systems/case_ACTIVSg10k/J.mtx \
    --rhs /datasets/power_system/nr_linear_systems/case_ACTIVSg10k/F.mtx \
    --repeat 300 --precision $prec | grep -E "factorize_ms|relative_residual"
done

# B=64 batched factor per system
for prec in fp16 fp16_wmma; do
  ./build/custom_linear_solver_run \
    --matrix /datasets/power_system/nr_linear_systems/case_ACTIVSg10k/J.mtx \
    --rhs /datasets/power_system/nr_linear_systems/case_ACTIVSg10k/F.mtx \
    --repeat 100 --batch 64 --batch-only --precision $prec | grep batch_factor_per_sys_ms
done
```
