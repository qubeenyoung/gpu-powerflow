8387 is structurally tiny-front dominated. `case13659pegase` is the remaining 13K middle point:
does it admit a paired B64/B256 Tensor Core speedup versus FP32?

## Decision

13K behaves closer to 8387 than to 25K/USA for the Tensor Core objective. Low-mid force-all exposes
more TC work than on 8387, but not enough for a stable paired B64/B256 `1.2x` result. The sampled
variants either shift only one batch size, slow the absolute path, fail factorization, or lose
accuracy.

Keep the accepted TC claim focused on 25K/USA. For 13K, further work should not be another local
mid-kernel toggle; it would need upstream panelization/order changes that create larger `nc,uc`
fronts, or a separate low-fill branch whose enabler is broader than per-front Tensor Core trailing.
